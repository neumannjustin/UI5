Step 26: Remote OData Service

So far we have only worked with local JSON data, but now we will access a real OData service. You can find many test services on the OData home page that we can use to connect our app to remote data. You will be surprised how few we have to change in order to make this work!

Coding

webapp/manifest.json
#!js{
  …
  "sap.ui5": {
	"_version": "1.1.0",
	"rootView": "sap.ui.demo.wt.view.App",

	"dependencies": {
	  "minUI5Version": "1.30",
	  "libs": {
		"sap.m": {}
	  }
	},
	"config": {
	  "invoiceLocal": "/Invoices.json",
	  "invoiceRemote": "/destinations/northwind/V3/Northwind/Northwind.svc/"
	},
	"models": {
	  "i18n": {
		"type": "sap.ui.model.resource.ResourceModel",
		"settings": {
		  "bundleName": "sap.ui.demo.wt.i18n.i18n"
		}
	  }
	}
  }
}

In the descriptor, we add a second custom configuration parameter that points to our remote data source. Due to cross-site scripting restrictions in JavaScript, we cannot connect to a remote URL from our local app. Instead we need to make sure our requests are sent via some proxy.

A proxy is simply a local script that connects to the remote server and returns the response to our app. In a productive scenario this would of course be a direct connection. We use the proxy to access the northwind test service from OData.org and fetch our invoices now from there instead of our local JSON data.

First, we need to configure a destination for a remote service in SAP Web IDE. When using such a destination, requests to our service will be proxied for us automatically. The URL shown in the sample code assumes there is a corresponding destination for the service configured.

Note

SAP Web IDE users can configure an HTTP destination instead of a proxy to access a remote service. For this additional configuration steps are required. See section Creating a Cloud Connectivity Configuration File in the SAP Web IDE documentation athttps://help.hana.ondemand.com/webide/frameset.htm for further details.

For all other development environments, you will need to use a local proxy.

webapp/Component.js
#!jssap.ui.define([
	"sap/ui/core/UIComponent",
	"sap/ui/model/json/JSONModel",
	"sap/ui/demo/wt/controller/HelloDialog",
	"sap/ui/model/odata/v2/ODataModel"
	
], function (UIComponent, JSONModel, HelloDialog, ODataModel) {
	"use strict";
	return UIComponent.extend("sap.ui.demo.wt.Component", {
		metadata : {
			manifest: "json"
		},
		init : function () {
			…

			// set invoice model - remote
			var oConfig = this.getMetadata().getConfig();
			var oInvoiceModel = new ODataModel(oConfig.invoiceRemote);
			this.setModel(oInvoiceModel, "invoice");
			// set dialog
			this.helloDialog = new HelloDialog();
		}
	});
});

You might wonder if that really was it. Yes, we simply have to load the OpenUI5 ODataModel class and exchange the local config parameter with the remote URL of the real northwind service. Since the test service has the same interface (except for the status, this one is not available in the real service) we can see all kinds of invoices in our app now.
